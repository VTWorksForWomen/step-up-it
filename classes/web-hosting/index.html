Servers

A server is a computer running software which is set up to accept requests from a client and give a response. There are many different types of servers.

  • Email
  • Web
  • FTP
  • List Server (like the step up it mailing list)

Web Servers

A web server is a specific type of server that accepts requests for a URL from a client (often a web browser) and returns a response (generally in HTML).

  • User enters URL http://www.vtworksforwomen.com/about-us into browser
  • Browser sends a request for the URL
  • Server receives a request for the URL
  • Server sends the text contents of an .html file in response
  • Browser receives the response
  • Browser finds any CSS or javascript or images in the HTML and sends requests to the server to download them
  • Browser displays the HTML to the user, applying any CSS and executing any javascript

TLDs
Top Level Domains

The end of a domain name (the part following the final period) is called the Top Level Domain (TLD).

Some TLDs are owned by countries (.co.uk), some of governmental (.gov) or education (.edu) and some are novelty (.guru, .photography).

  • .com (commercial)
  • .net (network)
  • .org (organization)
  • .biz (business)
  • .uk (united kingdom)
  • .photography

DNS: The Domain Name System

The Domain Name System, or DNS, is the system responsible for translating human readable domain names like google.com into unique IP addresses for devices on the internet.

This is the reason you can type google.com into your browser instead of needing to know the IP of the server where google.com is hosted, 216.58.219.206.

Web Hosting

Web hosting is renting a web server from a company that you can place all of the files for your website on. There are different types of web hosting.

  • Shared: Renting space on a server that also hosts other websites. Good for personal and small business websites. $4-$10/month.
  • Dedicated: Renting space on a server that hosts only your website. Required for security sensitive websites or sites with high traffic. $100+/month

Interpreted vs. Compiled

Interpreted

The computer can interpret and execute the code directly.

Examples: Javascript, PHP, Python, Perl and Ruby.

Compiled

First, the code is compiled into an executable program. Second, this program can be executed by the computer.

Examples: C, C#, Java

Interpreted vs. Compiled Advantages

Interpreted

  • Faster to write code
  • Cross-platform

Compiled

  • Faster to run code

REPL

Interpreted languages generally include tools called REPLs that allow you to enter a line of code and immediately see the result.

  1. Read: Read a line of code
  2. Evaluate: Interpret the line of code
  3. Print: Show the result
  4. Loop: Go back to step 1 and wait

Variables

Variables are named containers that can hold values.

Creating Variables

You create a variable by using the var keyword.
You decide on the name.


var taskName = "Grocery Shopping";
var taskIsComplete = false;
var taskPriority = 1;

Naming Variables

  • Variable names are case-sensitive.
  • Variable names should be unique
  • Variable names should only contain letters, numbers and certain special characters ($, _).
  • Some names are not allowed because they are reserved by the language like var and if
  • Naming your variables clearly is crucial to writing code that is easy to understand.

Changing Variable Values

You can change a variable's value after creating it by assigning a value using the equals sign. The code below creates a variable called taskName with the value "Grocery Shopping" and then changes the value to "Dog grooming".


var taskName = "Grocery Shopping";
console.log(taskName); // "Grocery shopping"
taskName = "Dog grooming";
console.log(taskName); // "Dog grooming"

Referencing Variables in Code

Once you have created a variable you can refer to its value in your code by using the variable name.


var age = 21;
var ageInMonths = age * 12;
console.log(ageInMonths); // Prints the result of 21 times 12

String Operators: +=

Functions

Functions are reusable pieces of code. Once we define a function we can use it over and over.

Functions often take some input values and return some output value.

For example an add function would take 2 numbers as input and return the sum as output.


var add = function(number1, number2) {
    return number1 + number2;
}

Calling Functions

To run the code inside a function, call the function using the function name followed by parentheses.

If the function takes any input values, they are placed inside the parentheses and separated by commas.

If the function returns any output value, you can store the value in a variable.


var add = function(number1, number2) {
    return number1 + number2;
}
var result = add(5, 10);
console.log(result); // 15

Anatomy of a Function: Arguments

A function can take input values, which are called arguments.

The add function we defined takes 2 arguments:
number1 and number2.


var add = function(number1, number2) {
    return number1 + number2; // 5 + 10
}

add(5, 10);

Anatomy of a Function: Return Values

A function can return output using the return keyword.

The code that calls the function can store the return value in a variable or use it directly.


var add = function(number1, number2) {
    return number1 + number2;
}

var sum = add(5, 10);
console.log(sum);
console.log(add(2, 3));
