Servers

A server is a computer running software which is set up to accept requests from a client and give a response. There are many different types of servers.

  • Email
  • Web
  • FTP
  • List Server (like the step up it mailing list)

Web Servers

A web server is a specific type of server that accepts requests for a URL from a client (often a web browser) and returns a response (generally in HTML).

  • User enters URL http://www.vtworksforwomen.com/about-us into browser
  • Browser sends a request for the URL
  • Server receives a request for the URL
  • Server sends the text contents of an .html file in response
  • Browser receives the response
  • Browser finds any CSS or javascript or images in the HTML and sends requests to the server to download them
  • Browser displays the HTML to the user, applying any CSS and executing any javascript

Domain Names
Human Friendly Addresses for Computers

A domain name is a human friendly unique string. Examples?

  • google.com
  • vsac.org
  • php.net
  • bbc.co.uk

TLDs
Top Level Domains

The end of a domain name (the part following the final period) is called the Top Level Domain (TLD).

Some TLDs are owned by countries (.co.uk), some of governmental (.gov) or education (.edu) and some are novelty (.guru, .photography).

  • .com (commercial)
  • .net (network)
  • .org (organization)
  • .biz (business)
  • .uk (united kingdom)
  • .photography

DNS: The Domain Name System

The Domain Name System, or DNS, is the system responsible for translating human readable domain names like google.com into unique IP addresses for devices on the internet.

This is the reason you can type google.com into your browser instead of needing to know the IP of the server where google.com is hosted, 216.58.219.206.

Domain Registrars

The first step in getting your website setup so someone can visit a domain like janeswidgets.com is registering the domain. This is done through an ICANN-accredited registrar. ICANN is the Internet Corporation for Assigned Names and Numbers, a non-profit company responsible for maintaining the databases that identify the location of all domain names in the world.

Web Hosting

Web hosting is renting a web server from a company that you can place all of the files for your website on. There are different types of web hosting.

  • Shared: Renting space on a server that also hosts other websites. Good for personal and small business websites. $4-$10/month.
  • Dedicated: Renting space on a server that hosts only your website. Required for security sensitive websites or sites with high traffic. $100+/month

Connecting a Domain to a Hosting Account

When you purchase a domain name from a registrar, you are generally also purchasing DNS hosting. This means that the registrar will host the information for the Domain Name System that helps resolve your domain name to the IP address of the server where it is hosted. By default, most registrars will set the DNS servers for a new domain to point to their DNS servers. This is fine if you purchase your domain and your web hosting from the same company. If you purchased your hosting from another company, they will tell you what their DNS servers are and you will need to update your domain at your registrar to point at those DNS servers.

Setting DNS Servers for Your Domain

If you purchased hosting from a different company than you purchased the domain from, you will need to log in to the registrar and set the DNS servers for your domain. The hosting company will generally email you a list of the DNS servers which will look something like.

  • dns1.namecheap.com
  • dns2.namecheap.com
  • dns3.namecheap.com

Every registrar has a different web-based interface for changing DNS servers, so you may need to search their site for "Change DNS Servers" to find instructions.

DNS Propagation

The master DNS database which holds all of the information on which IP hosts each domain name is maintained by ICANN and it can sometimes take 24-48 hours for changes to your DNS servers to propagate. This is especially true if you have previously visited the domain name, in which case your Internet Service Provider may have saved a local copy of the information of what IP to find the domain at. For this reason when buying a new domain its a good idea NOT to visit the domain your browser until you have configured the DNS servers.

Using a Proxy to View DNS Changes

If you have already visited a domain and want to change the DNS and see if your changes have taken effect, you can use a proxy service. A proxy allows you to enter a URL and it will load the page for you through another location. This can get around issues where your ISP has cached the IP address of the domain because the proxy will not use your ISP since it is in another physical location.

Uploading Your Website to a Web Server
using File Transfer Protocol (FTP)

FTP is a standard method for transfering files from one computer to another over the internet. FTP is commonly used to transfer website files from your computer to the web server you are renting to publish your website.

FTP Accounts

An FTP Account or FTP Credentials is the term of the login information to an FTP server. FTP credentials always include the following 3 pieces of information.

  • Username
  • Password
  • Server Address: Address of the server (e.g. ftp.mydomain.com)

FTP Client Software

An FTP Client is a piece of software that allows you to connect to an FTP server using your credentials (username/password). There are many pieces of free FTP client software.

  • Cyberduck
  • FileZilla
  • FireFTP

Uploading Files via FTP
The Web Root

The Web Root is the name of a folder on your web hosting server where files should be stored in order for them to be web accessible. Any files in this folder will be available via http://www.mydomain.com/filename.html. When you connect to your hosting account using FTP, you will see a list of files and folders. One of these folders is generally the webroot. The webroot folder is usually named one of the following things:

  • public_html
  • www
  • www.mydomain.com

The Home Page: index.html

Most popular pieces of web server software will automatically shows the file named index.html when the user requests a folder. When the user visit http://www.mydomain.com most web servers will look for a file named index.html in the web root folder.

Running a Web Server Locally

It is often convenient to run a web server locally on your computer so you can test your website. This is not required if your website is composed of simple html files, but if you use a server side language like PHP you will need a web server to interpret your PHP files into HTML. Depending on your operating system there are a number of free options for installing the popular web server Apache on your local computer.

Running the Apache Web Server
On Your Computer

  • Windows: XAMPP
  • Mac OS X: MAMP
  • Linux: LAMP
